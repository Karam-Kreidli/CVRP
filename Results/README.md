# Results Folder

## Why This Folder Exists

`Results/` stores analysis outputs generated by evaluation/comparison scripts.

This folder exists so experiment outputs are easy to inspect, share, and compare without rerunning heavy computations.

## What This Folder Contains (Current Snapshot)

```text
Results/
  Final_Full_Evaluation_Instances.csv
  Final_Full_Evaluation_Instances.md
  Final_Full_Evaluation_Report.md
  Final_Full_Evaluation_Summary.json
```

Current count in snapshot: `4` files.

## Typical Producers

Main producer:

- `Baseline/Infer.py` in `--baseline` mode with reports enabled.

It writes:

- `<stem>_Instances.csv`
- `<stem>_Summary.json`
- `<stem>_Report.md`

It can also write route files under `Solutions/` if solution export is enabled.

## File Roles

## `*_Instances.csv`

- per-instance RL vs baseline rows,
- includes NV, TD, score, delta, outcome, points, tie metadata, elapsed time.

## `*_Summary.json`

- structured aggregate metrics,
- win/loss/tie counts and rates,
- delta statistics,
- pairwise points totals,
- bootstrap robustness summary (when enabled).

## `*_Report.md`

- human-readable narrative summary,
- setup metadata,
- aggregate outcomes,
- top improvements/regressions,
- bootstrap interpretation.

## `*_Instances.md`

- markdown table export variant for easy sharing/review.

## How to Generate New Results

```bash
python Baseline/Infer.py \
  --instance_dir Data \
  --model_path Logs/Best_Model.pth \
  --baseline \
  --report_dir Results \
  --report_stem My_Experiment
```

## Why This Folder Is Separate from `Logs/`

- `Logs/` holds training-process artifacts.
- `Results/` holds experiment outcome reports and comparison outputs.

This separation keeps training internals and final evaluation deliverables clearly partitioned.
